API Structure   
1. Create a movie.
 --> POST --> /mba/api/v1/movies    here mba means movie bookig application.v1 is version 1.

 2. List all movies.
 --> GET  -->/mba/api/v1/movies      the url is called route  the GET route is used for listing all the movies.

 3. Get a particular movie(details about movie)
 --> GET --> .mba/api/v1/movies/:id         Here movie id(/:id) is a variable parameter.


 4. Get a movie based on name
 --> GET -->  /mba/api/v1/movies? name=harry    here ? --> is the query params and if want multiple query params then use &.

 5. Delete the movie
 --> DELETE --> mba/api/v1/movies/:id    for thos we pass the unique id we pass the unique id this will be delete that.

 6. Update the movie.
--> PUT --> /mba/api/v1/movies/:id     here the movie id will be send that particular id will be updated.
